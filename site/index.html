wmux

Le terminal conçu pour Claude Code les agents IA le multitasking Windows

Application Windows basée sur Electron. Onglets verticaux, notifications quand les agents ont besoin d'attention, panneaux divisés, navigateur intégré et une API pipe pour l'automatisation.

Claude Code Plugin
wmux-orchestrator

Lancez plusieurs agents Claude Code en parallèle depuis une seule commande. Le plugin analyse votre codebase, découpe la tâche en sous-tâches indépendantes, et les distribue en vagues avec gestion des dépendances, zones de fichiers isolées et revue automatique en fin de cycle.

Installation
$ claude plugin install wmux-orchestrator@wmux-orchestrator
Voir sur GitHub Documentation

Fonctionnalités

  • Intégration passive Claude Code : wmux observe Claude Code sans modifier son comportement. Un proxy CDP sur localhost:9222 permet au navigateur intégré de montrer chaque action en temps réel. Les hooks auto-configurés rapportent l'activité des agents et outils dans la barre latérale. Zéro configuration.
  • Navigateur intégré : ouvrez un navigateur à côté de votre terminal avec une API scriptable. Quand Claude Code navigue via chrome-devtools-mcp, chaque page, clic et formulaire est visible dans le panneau navigateur.
  • Anneaux de notification : les panneaux s'illuminent en bleu quand les agents ont besoin d'attention. Notification Windows toast, flash barre des tâches, et centre de notifications intégré.
  • Onglets verticaux : la barre latérale affiche la branche git, le répertoire de travail, les ports, le nombre d'agents, le statut PR et le texte de notification. Double-clic pour renommer.
  • Panneaux divisés : divisions horizontales et verticales dans chaque onglet. Ctrl+D pour diviser à droite, Ctrl+Shift+D pour diviser en bas.
  • Indicateurs d'activité : orange pulsant = en cours, vert = terminé, rouge = interrompu. Visible d'un coup d'œil dans la barre latérale.
  • Sessions sauvegardées : sauvegardez vos splits, répertoires et URL navigateur. Restauration automatique au démarrage.
  • Scriptable : CLI et API named pipe (\\.\pipe\wmux) pour l'automatisation et le scripting. Protocole JSON-RPC v2 compatible avec cmux.
  • Natif Windows : ConPTY pour l'émulation de terminal, notifications toast Windows, flash barre des tâches, title bar overlay native.
  • Accélération GPU : propulsé par xterm.js avec rendu WebGL pour un affichage fluide.
  • Thèmes compatibles : importez vos thèmes depuis Windows Terminal ou Ghostty. 450+ thèmes Ghostty inclus.
  • Raccourcis clavier : raccourcis complets pour les espaces de travail, les divisions, le navigateur et plus.

Questions fréquentes

wmux est un fork Windows de cmux. cmux est une application macOS native (Swift + AppKit + Ghostty). wmux reproduit la même expérience sur Windows en utilisant Electron + xterm.js + ConPTY. Le protocole CLI/pipe est compatible entre les deux.

Windows 10 et 11 uniquement (x64). Pour macOS, utilisez cmux.

wmux est optimisé pour Claude Code avec intégration passive automatique (hooks, CDP proxy, CLAUDE.md). Tout agent ou programme en ligne de commande peut être utilisé dans les terminaux wmux.

Les scripts d'intégration shell détectent quand une commande se termine ou est interrompue. Le panneau reçoit un anneau bleu, le badge sidebar s'incrémente, et une notification Windows toast apparaît. Supporte OSC 9/99/777, la CLI wmux notify, et la détection d'inactivité.

Windows Terminal est un excellent émulateur de terminal, mais il n'a pas de système de notification, pas de visibilité sur l'activité des agents, pas de navigateur intégré, et pas de métadonnées live dans les onglets. wmux est conçu spécifiquement pour superviser des agents IA.
